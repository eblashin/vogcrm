Документ "Интерес"

Ведение взаимодействий и их планирование – одна из основных задач, решаемых при помощи «1С:CRM». Весь учет взаимодействий и документов, связанных с ними, ведется в едином месте – в документе «Интерес». Сценарии работы в документе "Интерес" представлены в разделе "Главное -> Помощь -> Ведение взаимодействий, их планирование".

Назначение

Основной документ конфигурации, предназначенный для управления продажами и взаимодействиями с клиентами. Телефонные звонки, встречи, письма, коммерческие предложения, счета и др. привязываются к какому-либо Интересу клиента, так как эти объекты служат определенной цели общения с клиентом. Это необходимо для сохранения истории общения с Клиентом.

Новый Интерес может быть создан в следующих случаях:
• при первом обращении Клиента с потребностью в каком-либо продукте компании (нет ни одного активного Интереса);
• при первом общении менеджера с Клиентом по продаже какого-либо продукта;
• при появлении новой потребности у Клиента, не соответствующей уже имеющимся активным Интересам.

За весь цикл работы с Интересом (от его создания до завершения) работу по нему могут выполнять разные менеджеры, но на каждом этапе работу с Интересом должен вести только один менеджер по продажам, назначенный по нему ответственным.


Важно! Запрещена запись Интереса без запланированного следующего взаимодействия.

 

Все исходящие взаимодействия с клиентом можно осуществлять только из Интереса. Во всех остальных формах возможности регистрации контактов с клиентами нет. Информация по всем взаимодействиям в рамках обработки Интересов, по которым менеджер по продажам является ответственным, отображается в АРМ «Мои дела».

При необходимости любой сотрудник компании может отслеживать состояние взаимодействий с Клиентом по тому или иному Интересу, взяв их на контроль.

Принцип работы с Интересами устроен таким образом, что перевести Интерес в новое состояние можно только при условии, что по текущему взаимодействию указан результат.

В новом планируемом взаимодействии будут указаны те же состояние интереса и сумма ожидаемой выручки, что и в предыдущем взаимодействии. Эти значения можно изменять только после записи результата по предыдущему взаимодействию.

 

Работа по Интересу ведется по вышеописанному порядку до тех пор, пока состояние последнего взаимодействия Интереса не примет одно из следующих значений:
• «Выполнение договора (100%)»;
• «Интерес закрыт (100%)»;
• «Интерес потерян (100%)».

С целью ведения дополнительной аналитики для интересов клиентов реализован механизм тегов. Теги задаются внутри подразделения, для логической быстрой отметки. Ведение аналитики с помощью тегов осуществляется в разрезе подразделений. Для каждого подразделения задаются такие теги, которые используются для маркировки документов продажи. Каждому подразделению необходимо вести свой набор тегов. При работе с Интересом пользователь может добавлять в него теги, но только из списка доступных для данного подразделения.

 

Создать новый тег для подразделения может только руководитель этого подразделения.

Вводится на основании: «Интерес», «Клиенты».

 

Объекты, вводимые на основании: «Взаимодействие», «Интерес», «Мероприятие», «Рассылка электронных писем», «Счет на оплату», «Коммерческое предложение», «Электронное письмо исходящее».

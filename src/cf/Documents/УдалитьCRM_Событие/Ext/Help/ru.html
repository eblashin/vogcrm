Документ "Событие"

Назначение

Документ предназначен для регистрации событий.

События делятся по своему назначению на следующие виды:

- телефонный звонок;

- личная встреча;

- прочее;

- почтовое письмо;

- электронное письмо.

Событие каждого вида рассматривается как входящее или исходящее. Этот параметр устанавливается в верхней части карточки события.

Событие может быть создано с клиентами и контактными лицами. Выбор производится, соответственно, в полях Клиент и Контактное лицо.

В поле Место указывается место проведения события. Это может быть внесенный в справочник «Помещения»  адрес клиента, контактного лица или помещения организации. Адрес также можно ввести вручную.

Событие может иметь несколько состояний:

- запланировано - событие должно произойти в будущем;

- в работе - событие происходит в настоящий момент;

- отменено - событие не произошло; выбрав это состояние, пользователь может указать в поле Причина отказа  причину отказа от проведения данного события, выбрав ее из справочника «Причины отказов»;

- завершено - событие произошло; после установки этого состояния при записи документа система предлагает запланировать следующее событие по данному клиенту.

В поле Важность можно установить приоритет обработки событий. Важность может быть определена как высокая, средняя или низкая.

Для каждого события указывается интервал времени, отведенный на его выполнение: дата начала и окончания события.

 

В карточке документа Событие могут присутствовать закладки:

- Описание;

- Интерес клиента;

- Участники;

- Дополнительно;

- Оповещения.

 

На закладке Описание в верхнем поле вводится описание события. Также на этой закладке размещено поле Результат.

 

На закладке Интерес клиента можно указать список товаров и услуг с ценами, связанных с данным событием.

Если в данном событии участвует несколько клиентов, контактных лиц или сотрудников Вашей компании, то их список можно указать на закладке Участники.

В левой части закладки вводится список участников -  контактных лиц клиента; в правой части вводится список участников - сотрудников компании.

 

На закладке Дополнительно можно заполнить поля Подразделение и Основание - документ, на основании которого было введено данное событие. Здесь же указываются трудозатраты в часах, автоматически рассчитываемые как промежуток времени между началом и окончанием события.

 

На закладке База знаний можно подобрать требуемый текст решения и добавить его текст в поля Описание и Результат.

В верхней части страницы расположены поля полнотекстового поиска решения в базе знаний:

- в первом поле можно выбрать, в каких реквизитах базы знаний выполнять полнотекстовый поиск;

- установка флажка "(включая подстроку)" включает поиск по вхождению подстроки;

- в следующем поле указывается строка поиска;

- установка флажка "искать по проекту" включает условие поиска только тех решений, которые относятся к проекту документа.

В левой части расположена таблица решений, которые были использованы в процессе заполнения данного документа или которые были найдены последним поиском по базе знаний.

У решений, которые были использованы при заполнении документа, проставлен флажок "Использовать". При сохранении данного документа сохраняется информация об отмеченных решениях, и при повторном открытии документа они загружаются и отображаются в таблице решений.

В правой части расположено поле, в которое выводится описание решения: текст вопроса и все ответы на него, отсортированные по следующим условиям:

- первый всегда отображается вопрос;

- если для решения в таблице решений указан выбранный ответ, то первый выводится он;

- если решение найдено полнотекстовым поиском, то следующие выводятся ответы, в которых присутствует строка поиска;

- последними выводятся остальные ответы, отсортированные по своему состоянию.

Если решение найдено полнотекстовым поиском, то в поле описания решения желтым цветом выделяются найденные строки поиска, для перемещения по которым сверху поля на командной панели расположены кнопки перехода по найденным строкам.

На этой же панели расположены кнопки добавления текста в поля Описание и Результат данного документа. Для того чтобы добавить нужный текст в любое из этих полей необходимо или выделить текст, который нужно добавить или поместить курсор на позицию внутри нужного текста и нажать кнопку "Добавить в ...". В конец текста поля добавляется, если нужный текст выделен, то только выделенный текст, иначе весь текст, относящийся к данному вопросу или ответу.

На форме документа Событие пользователь может указать к какому проекту/бизнес-процессу относится данное событие. Данное значение выбирается в раскрывающемся меню кнопки над полем Тема.

 

Взаимодействие с бизнес-процессами.

Старт бизнес-процесса из формы документа.

 

Если в документе выбран проект, в котором в свою очередь, указана карта маршрута, то

на командной панели документа, появится кнопка «Начать <имя выбранной карты маршрута>».

При нажатии на эту кнопку будет открыта форма создания нового бизнес-процесса.

 

 

Связывание документа с ранее созданными бизнес-процессами.

 

Если используется подсистема бизнес-процессов, то документ можно связать с

текущим бизнес-процессом клиента.

 

Если в документе указан клиент и по клиенту есть не завершенный  бизнес-процесс, то он подставляется в поле «Бизнес-процесс»,

а в поле «Задача» подставляется текущая (не выполненная) задача бизнес-процесса.

 

Если же не завершенных бизнес-процессов несколько, то выбор предоставляется пользователю на его усмотрение.

 

При вводе документа на основании другого документа, связанного с каким- либо бизнес-процессом, информация в поля «Бизнес-процесс» и «Задача»

будет подставляться данными о бизнес-процессе, с которым связан документ основание.

 

Связывание с бизнес-процессом можно произвести, нажав кнопку «Связать», если у пользователя в персональных настройках включено

«Автоматически связывать взаимодействия с бизнес-процессами», то кнопки «Связать» на форме не будет, а связывание будет происходить в момент записи документа.

 

После того, как документ связан с бизнес-процессом, этот документ можно будет увидеть

в списке объектов по бизнес-процессу и в задаче (история выполнения и список действий).

Создается в подсистемах:

«Календарь и почта»

«Работа с клиентами»

«Рабочий стол»

Вводится на основании:

«Маркетинговые кампании», «Счет на оплату покупателю», «Коммерческое предложение», «Событие», «Электронное письмо входящее», «Электронное письмо исходящее».

Объекты, вводимые на основании:

«Рассылка электронных писем», «Событие», «Счет на оплату покупателю», «Коммерческое предложение».

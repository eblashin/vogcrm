Обработки "Мастер проверки клиентской базы"

Назначение

Мастер предназначен для регулярного контроля за качеством клиентской базы. Мастер включает решение следующих задач:

  • Мастер заполнения контактной информации: решает задачу по заполнению нужных видов контактной информации(emal, телефоны, адреса и т.д.). Мастер позволяет выбрать один или несколько видов контактной информации, автоматически проанализировать по всей клиентской базе и получить клиентов у которых частично и/или полностью не заполнена контактная информация. После получения клиентов предлагается выбрать один из вариантов, каким образом назначить исполнителей для решения задачи. После выбора исполнителей, мастер формирует для каждого исполнителя поручение, в котором отражается список всех назначенных клиентов для обработки. В данном списке решение автоматически отслеживает введена ли недостающая информация по клиенту.
  • Мастер заполнения свойств клиентов: решает задачу по первоначальному заполнению сегментации клиентов и/или заполнению важных свойств клиентов при определении таковых. Мастер позволяет выбрать один или несколько реквизитов клиента и их дополнительных реквизитов (важных реквизитов, чем отличаются клиенты друг от друга относительно работы с ними), автоматически проанализировать по всей клиентской базе не заполненные реквизиты и получить список таких клиентов. После получения клиентов предлагается выбрать один из вариантов, каким образом назначить исполнителей для решения задачи. После выбора исполнителей, мастер формирует для каждого исполнителя поручение, в котором отражается список всех назначенных клиентов для обработки. В данном списке решение автоматически отслеживает введена ли недостающая информация по клиенту.
  • Мастер поиска задвоенных клиентов: решает задачу выверки клиентской базы, когда клиентская база только формируется или используется много филиалов/отделов работающих с клиентами. Мастер позволяет по выбранным показателям, а именно "Наименование", "Наименование полное", "ИНН", "Адрес", "Email", "Телефон", определить клиентов у которых буду совпадать указанные показатели. После получения клиентов предлагается выбрать один из вариантов, каким образом назначить исполнителей для решения задачи. После выбора исполнителей, мастер формирует для каждого исполнителя поручение, в котором отражается список всех назначенных клиентов для обработки. 
  • Мастер контроля потенциала клиентов: решает задачу по заполнению/обновлению значения потенциала клиентов. Мастер позволяет выбрать критерии поиска клиентов для заполнения/обновления значения потенциала - с незаполненным потенциалом или если были изменены реквизиты, влияющие на потенциал. После получения клиентов предлагается выбрать один из вариантов, каким образом назначить исполнителей для решения задачи. После выбора исполнителей, мастер формирует для каждого исполнителя поручение, в котором отражается список всех назначенных клиентов для обработки. В данном списке решение автоматически отслеживает заполнено ли значение потенциала по клиенту. 
  • Мастер контроля регистрации клиентов: решает задачу по заполнению клиентов в событиях, в которых клиент заполнен не был. Мастер позволяет установить отбор, ограничивающий список событий с незаполненным клиентом. После получения событий предлагается выбрать исполнителей для решения задачи. После выбора исполнителей, мастер формирует для каждого исполнителя поручение, в котором отражается список всех назначенных событий для обработки. В данном списке решение автоматически отслеживает заполнен ли клиент в событии.

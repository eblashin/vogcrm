Настройки параметров учета

На формах настроек параметров учета располагаются страницы, на которых задаются значения настроек и параметров, которые являются общими для всех пользователей системы и определяют используемый функционал программы.

Страница «CRM: Общие настройки»

На странице устанавливаются общие параметры системы: основной календарь, интервал рабочего времени, использование подсистемы "Управление проектами", а также устанавливаются общие настройки по продажам, клиентам и журналу документов.

Страница «CRM: Оповещения»

На странице устанавливаются настройки использования напоминаний и оповещений, в том числе по e-mail и смс.

Страница «CRM: Работа с SMS»

На странице устанавливается использование подсистемы работы с сообщениями СМС, задаются провайдер, логин и пароль. Без этих данных невозможно установить сеанс работы с провайдером СМС.

  • Провайдер.  Провайдер СМС, через котрого будет выполняться отправка сообщений.
  • Логин.  Имя пользователя.
  • Пароль.  Пароль пользователя.

Настройки провайдера «SMS4B.RU»

  • Получать только полные сообщения.  Флаг означает, что при получении входящих сообщений, будут приходить те сообщения, которые полностью получены сервером SMS4B.
  • Дата последнего входящего сообщения.  Дата, с которой получаются входящие сообщения. Не рекомендуется её изменять. 
  • Робот регламентных заданий. Пользователь, под которым будет выполнятся регламентное задание по получению, отправке сообщений.


После ввода учетных данных рекомендуется проверить их, выполнив подключение к серверу. Для этого нажмите на кнопку "Подключиться в серверу SMS4B и заполнить список отправителей".

Настройки документа «SMS сообщение»

В этом разделе задаются значения по умолчанию для подстановки в документ "SMS сообщение".

  • Номер отправителя по умолчанию. Номер отправителя по умолчанию выбирается из списка доступных номеров той учетной записи, имя пользователя и пароль которой указаны в настройках.
    Внимание! Список будет заполнен только после ввода корректных данных учетной записи и подключения к серверу SMS4B.»
  • Срок жизни SMS (в часах). Количество часов, в течении которых сервер будет пытаться отослать сообщение. Если сообщение не удастся отправить по истечению срока жизни, оно будет считаться "просроченным".
  • Не отправлять в период с ... по ... . Задается период, в пределах которого сообщения не должны доставляться. Например, ночные часы.

Раздел «Настройка прокси-сервера»

В данном разделе указываются настройки прокси-сервера. Указывать настройки нужно только в том случае, если в организации доступ к интернет реализован через прокси-сервер.

В случае, если для авторизации на прокси-сервере не требуется указывать имя пользователя и пароль, может быть достаточно параметров, указанных в Internet Explorer. В этом случае параметры прокси-сервера можно не вводить. Если же необходима авторизация или сервер предприятия работает в среде Linux, необходимо заполнить параметры и обязательно создать файл настроек прокси-сервера. Для этого нажмите кнопку "Создать файл настроек прокси на сервере 1С:Предприятия".

  • Прокси-сервер. IP-адрес или имя прокси-сервера.
  • Порт. Порт прокси-сервера.
  • Пользователь. Имя пользователя для авторизации на прокси-сервере.
  • Пароль. Пароль пользователя для авторизации на прокси-сервере.

В случае использования клиент-серверной базы файл автоматически будет создан на сервере 1С:Предприятия. В случае использования файлового режима необходимо создать (скопировать) файл настроек на всех компьютерах, где подразумевается работа в сервером SMS4B. 
В случае, если система не сможет создать файл автоматически (например, из-за отсутствия необходимых прав доступа), необходимо обратиться с системному администратору и скопировать его вручную в каталог программы 1С:Предприятия в подкаталог "Conf".
Описание формата файла настроек приведено в главе "Приложение В. Формат файла настройки прокси по умолчанию inetcfg.xml" книги «1С:Предприятие 8.1. Конфигурирование и администрирование».

Страница «CRM: СофтФон»

На странице устанавливается использование подсистемы работы с телефонией "СофтФон", выполняется подключение к серверу СофтФон и заполняются параметры и настройки работы с данной подсистемой.

Страница «CRM: Сторонние сервисы»

На странице выполняются настройки параметров взаимодействия информационной базы с внешними приложениями и сервисами (MS Outlook, MS Exchange, ЛИК:Бизнес, Сервисы Google).

Обработки "Обмен данными с 1С:Бухгалерия"

Назначение

Выгрузка данных из «1С:Бухгалтерия»

 Для обмена данными необходимы следующие файлы, которые находятся в подкаталоге ExchBuh каталога шаблонов:

Обмен с 1C:Бухгалтерия предприятия 8 (1.6/2.0) - универсальная обработка обмена данными "CRMЕxchange.epf". 

Обмен с  Бухгалтерский учет 7.7 - V77Exp.ert, внешняя обработка выгрузки данных во внешний файл, V77Imp.ert, внешняя  обработка  загрузки  данных из внешнего файла, Acc77_CRM82.xml – правила конвертации данных из конфигурации "Бухгалтерский учет 7.7" в конфигурацию "1С:CRM СТАНДАРТ".

Выгрузка счетов на оплату

   Команда предназначена для выгрузки данных о продажах, за указанный период и по выбранной организации.  Продажи выгружаются в виде документа «Счет на оплату».
В поле ‘Тип базы приемника’ указывается в какую версию конфигурации «1С:Бухгалтерия» будут впоследствии загружены данные. В поле ‘Имя файла данных’ выбирается  файл в который будут сохранены выгруженные данные.

 Импорт справочников из «1С:Бухгалтерия»

Команда предназначена для загрузки данных выгруженных из программы 1С:Бухгалтерия (данные выгружаются внешней обработкой CRMExchange.epf или V77Exp.ert в зависимости от редакции 1С:Бухгалтерии).  Из бухгалтерии можно выгрузить номенклатуру, контрагентов и цены номенклатуры.

Загрузка взаиморасчетов из «1С:Бухгалтерия»

Команда предназначена для загрузки  данных о взаиморасчетах с контрагентами из «1С:Бухгалтерия». Например: из CRM с помощью команды «Выгрузка счетов на оплату» выгружаются  в бухгалтерию счета на оплату (за указанный период и по выбранной организации). Далее счета загружаются в 1С:Бухгалтерию. В бухгалтерии проводятся документы оплаты и реализации.  Из базы бухгалтерии в виде файлов данных выгружаются данные о реализации и оплатах (данные выгружаются внешней обработкой CRMExchange.epf или V77Exp.ert в зависимости от редакции 1С:Бухгалтерии). 


Распределение взаиморасчетов по продажам CRM
   

После того как файл данных будет выбран в поле обработки ‘Файл обмена’ и загружен, информация о реализации и оплатах отобразится на закладках ‘Реализация’ и ‘Авансы’.  Если в файле данных была информация о реализации, то она попадет в табличную часть ‘Документы реализации’, информация об оплате реализации или о погашении авансовых платежей реализацией будет выведена в табличной части ‘Документы оплаты’. Информация об авансах попадет в табличную часть ‘Документы оплаты’ на закладке ‘Авансы’.

Важно! если в 1С Бухгалтерии 1.6  выбран вариант ведения учета расчетов с контрагентами только по договорам, либо используется 1С Бухгалтерия версии 7.7, тогда при загрузке поступившей оплаты в CRM автоматически подбираются продажи (по принципу FIFO).

Авансовые платежи как и оплату, можно распределить на конкретный документ продажи. Для данных о зачтенных авансах, поступивших из бухгалтерской базы,  подбираются неоплаченные документы продажи, также как и для данных об оплатах. При дальнейшем разнесении по документам продажи, сумма аванса (если он был распределен) в документе (на который она была распределена) корректируется в меньшую сторону на зачтенную сумму.

Пример: На первое число месяца поступил авансовый платеж от покупателя в размере 1000 руб. и данные об авансе были загружены в CRM Стандарт и распределены на документ продажи (допустим документ продажи №1). Далее в течение недели в бухгалтерии была сформирована продажа на сумму 500 руб. и был зачтен аванс на ту же сумму в  500 руб.  Данные также были загружены в CRM Стандарт. При распределении данных о факте зачета аванса, на какой либо документ продажи, информация об авансе в документе продажи №1, на который он был распределен, будет скорректирована (уменьшена на 500 руб.).

 

Сопоставление загруженных данных продажам.

      
Если обмен происходит с «Бухгалтерией предприятия 2.0»  и реализация в бухгалтерии была создана на основании счета загруженного из базы CRM, то обработка автоматически сопоставит загруженные данные с продажами. В случае, когда обмен происходит с  «Бухгалтерией предприятия 1.6» или «Бухгалтерией 7.7» можно воспользоваться командой ‘Подобрать продажи’ в меню ‘Автозаполнение’.  
 

    Внимание! Следует учитывать, что при автоматическом подборе обработка ищет продажи удовлетворяющие следующим условиям:

  1. Совпадают:
    •  Организация;
    •  Контрагент;
    •  Договор контрагента;

   2.   Остаток по отгрузке больше ‘0’;

   3.   Дата начала продажи меньше или равна дате документа реализации;

   4.   Номенклатура реализации должна присутствовать в продаже;

 

   Если есть такая продажа, то она будет выбрана и подставлена в поле ‘Продажа’ табличной части ‘Документы реализации’,  а так же в аналогичное поле табличной части ‘Документы оплаты’ для всех документов оплаты связанных с данной реализацией. Подбор продаж для авансов делается таким же способом.

  При ручном выборе документа продажи можно предварительно просмотреть состав текущего документа реализации по кнопке ‘Показать состав’. Когда продажа будет выбрана, она автоматически подставится во всех связанных с документом реализацией оплатах.

      Создание продаж на основании загруженных данных
 
   Когда реализация в бухгалтерии была создана не на основании счета, загруженного из базы CRM, может возникнуть ситуация, что нет продажи, с которой можно сопоставить данные о загруженной реализации из бухгалтерии. Для этого в подменю ‘Автозаполнение’ есть кнопка ‘Создать продажи’.
   Команда открывает форму со списком всех загруженных документов реализации, у которых не заполнено поле ’Продажа’.
   Отметив флажками те документы, по которым требуется создать продажи, по кнопке ‘Создать’ будут созданы документы продажи и автоматически подставлены для указанных документов реализации и связанных с ними документов оплат.

      Загрузка оплат по ранее разнесенным реализациям и авансам.
     
   Может возникнуть ситуация, когда загруженный документ оплаты связан с документом реализации, который был ранее загружен и распределен по продажам в CRM. Такой документ реализации будет отображен в списке  ‘Документы реализации’ (шрифт текста строки будет жирным и зеленого цвета), а в поле ‘Продажа’ строки документа оплаты подставится продажа, на которую ранее была распределена реализация.
   Также загруженный документ оплаты может быть связан с ранее загруженным и распределенным авансом. В таком случае строка документа оплаты будет выделена шрифтом синего цвета, а поле ‘Продажа’ будет подставлена продажа, на которую ранее был распределен аванс.

    

      Распределение реализации и оплат по продажам.

 После того как были сопоставлены данные по реализации, оплатам и авансам с продажами  необходимо нажать кнопку  ‘Распределить’.  Обработка распределит данные о реализации и оплате по выбранным продажам.

Справочник "Клиенты"

Назначение

Для хранения сведений обо всех клиентах организации используется справочник «Клиенты».
В карточке клиента, которая является элементом справочника «Клиенты», вводится информация о клиенте, редактируются данные.
Карточку клиента можно создать:
• при регистрации нового Интереса;
• из формы Интереса потенциального клиента, созданного при помощи Мастера принятия обращения;
• непосредственно в справочнике «Клиенты».

Список клиентов

Список клиентов предназначен для быстрого поиска клиента по изветсным данным и просмотра имеющейся информации по нему. Для поиска клиента используется поле "Поиск (Ctrl+F)". При этом выполняется поиск по видимым текстовым полям в списке клиентов. Для поиска по другим уникальным полям необходимо воспользоваться командой "Детальный поиск". В этом случае будет доступен поиск по следующим данным:

- ИНН: поиск выполняется только по реквизиту клиента "ИНН"

- Телефон: поиск выполняется по всем видам контактной информации с типом "Телефон", указанных как в карточке клиента, так и его контактных лиц

- Email: поиск выполняется по всем видам контактной информации с типом "Адрес электронной почты", указанных как в карточке клиента, так и его контактных лиц

- Прочая контактная информация: поиск выполняется по всем видам контактной информации кроме типов "Адрес электронной почты" и "Телефон", указанных как в карточке клиента, так и его контактных лиц

- Наименование: поиск выполняется только по наименованию клиента

- Контактное лицо: поиск выполняется только по наименованию контактного лица

Закладка «Портрет клиента»
На закладке «Портрет клиента» находятся следующие реквизиты:
• «Вид клиента». Юридическое лицо (Компания) или физическое лицо. В зависимости от вида клиента изменяется набор заполняемых полей на закладках «Реквизиты» и «Дополнительные реквизиты».
• «Наименование». В этом поле указывается краткое наименование клиента.
• «Полное наименование». Указывается полное наименование клиента.
• «Потенциал клиента». Нажатие на гиперссылку «Потенциал…» открывает закладку «Потенциалы» карточки клиента, рассмотрено далее.
• «Информация о задолженности». По данной гиперссылке открывается ведомость «Взаиморасчеты с клиентами».
• «Бизнес-регион»;
• «Типы отношений» — покупатель, поставщик, конкурент и др.
• «Важность» — обычный клиент, особо важный клиент и др.;
• «Статус работы» — работаем, не работаем, потерянный клиент и др.;
• «Отрасль»;
• «Сегмент рынка»;
• «Численность сотрудников»;
• «Численность рабочих мест».

По кнопке   можно получить информацию о клиенте из сервиса ЛИК:Бизнес (подробное описание содержится в разделе «Органайзер» решения, группа «Помощь» ).
На закладке «Портрет клиента» также можно поместить изображение клиента (фото, логотип и т. п.).


Закладка «Контакты»
На этой закладке представлена контактная информация клиента (адреса, номера телефонов, электронные адреса, web-сайты) и список контактных лиц клиента с информацией о них.
Важно!
Если при заполнении контактной информации не хватает позиций для ввода, следует в разделе решения «Настройки», подраздел «Общие настройки», «Контактная информация», пункт «Виды контактной информации», указать дополнительный вид (или несколько видов) контактной информации для клиента.


Закладка «Реквизиты»
Закладка «Реквизиты» предназначена для ввода информации, необходимой для оперативной оценки клиента и последующей работы с ним. Это такие реквизиты как Банковский счет, ИНН, ОКПО и др.
Также на этой закладке указывается ответственный менеджер по данному клиенту, его подразделение, договор с клиентом, актуальный источник привлечения клиента и др. В поле «Перевод звонков на» можно указать сотрудника, на которого будут переводиться звонки от данного клиента, в случае отсутствия/занятости основного менеджера клиента. Данный реквизит доступен для заполнения, если включена настройка «Использовать сервис Интеграции с телефонией» в разделе «Настройки», пункт «Интеграция с телефонией».

Закладка «Дополнительные реквизиты»
Закладка «Дополнительные реквизиты» содержит дополнительные реквизиты клиента. Это могут быть как сертификаты на выполнение определенных видов работ, так и дополнительная информация о том, что клиенты, например, могут быть партнерами по реализации определенных товаров компании. Также дополнительными реквизитами могут быть и другие важные для компании отличительные особенности клиента (реквизиты обычно определяются маркетологами в процессе сегментации клиентской базы). В диалоговой части данной закладки отображаются дополнительные реквизиты по клинту, заданные в настройках системы (раздел решения «Настройки», «Общие настройки», пункт «Дополнительные реквизиты и сведения», «Дополнительные реквизиты»). В области «Дополнительная информация» данной закладки отображаются дополнительные сведения клиента. Задаются в настройках системы (раздел решения «Настройки», группа «НСИ», пункт «Набор типов дополнительной информации»). Отличие дополнительной информации от дополнительных реквизитов в том, что в дополнительной информации могут быть указаны одновременно несколько значений по одному и тому же свойству. Например, могут быть указаны сведения по имеющимся у клиента сертификатам на различные виды работ.


Закладка «Уровни доступа»
Закладка «Уровни доступа» содержит сведения об уровнях доступа к клиенту. Уровни доступа ограничивают доступ к информации на уровне записей в программе. Информация по клиенту доступна пользователям, у которых уровень доступа не ниже уровня доступа клиента. Для того, чтобы в программе применялось ограничение на уровне записей, необходимо в разделе решения «Настройки», пункт «Настройки пользователей и прав», установить флажок «Использовать ограничения доступа для клиентов».


Закладка «Потенциалы»
Настройка логики расчета потенциалов задается в форме подразделения (справочник «Подразделения предприятия» ).
Данный механизм доступен только для КОРП версии поставки.

В форме настройки расчета потенциалов необходимо задать формулу расчет потенциала клиента на основании реквизитов портрета клиента. Для нужных реквизитов необходимо задать веса для каждого из его значений. Согласно введенной формуле, в зависимости от значения показателя для конкретного клиента, в алгоритм расчета подставляются конкретные веса выбранных реквизитов и рассчитывается итоговый потенциал.
Потенциалы клиентов рассчитываются автоматически регламентным заданием "Расчет потенциала клиентов».

 

В списке справочника «Клиенты» применяется графическое оформление по проценту заполненности портрета и контактной информации по клиенту и его контактным лицам. При расчете процента учитываются заполненные данные по портрету клиента, по основным видам контактной информации, а также по портрету контактных лиц и их основным видам контактной информации. При расчете процента для контактного лица клиента учитываются заполненные данные по портрету контактного лица, по основным видам контактной информации.

Справочник "Сегменты клиентов"

Справочник "Сегменты клиентов" предназначен для хранения списка всех сегментов клиентов, определенных в предприятии. С помощью сегмента клиентов производится дополнительная классификация клиентов, которая используется при определении правил взаимодействий и типовых условий продаж, а также при проведении маркетингового анализа. Разделение клиентов на сегменты позволяет:

  • Понимать с каким клиентом мы имеем дело (его особенности) 
  • Проводить по каждому сегменту (группе сегментов) клиентов отдельные маркетинговые кампании.

Сегмент – это набор признаков и критериев, удовлетворяющих правилам, определенным для сегмента. Он предоставляет информацию по элементам этого набора в виде, соответствующем цели создания сегмента. Сегменты допускают пересечения (один элемент может входить в несколько сегментов). Правила формирования и формы представления сегментов могут использовать произвольную информацию, хранимую в системе. Поддерживаются три способа формирования сегментов.

  • Динамический – набор элементов не хранится в таблицах базы данных, хранятся только правила его формирования. Состав сегмента формируется автоматически при вызове состава сегмента из карточки сегмента.
  • Периодическое формирование – набор элементов хранится в таблицах базы данных. Состав сегмента формируется при нажатии на кнопку "Сформировать" или при выполнении регламентного задания "Формирование сегментов" по расписанию, определенному пользователем;
  • Формирование «вручную» – пользователь может добавить или удалить элементы набора сегмента (при этом предоставляется возможность предварительно сформировать набор элементов по правилам, определенным для сегмента).

Правила формирования сегмента задаются непосредственно в форме сегмента в отдельном диалоговом окне, которое открывается при нажатии на кнопку "Настройка". Пользователь может использовать шаблоны настроек (кнопка "Выбрать шаблоны"), с помощью которых формируются правила формирования сегмента клиентов. Для сегмента клиентов предусмотрены следующие шаблоны настроек правил формирования сегмента.

  • Обычная схема. С помощью этой схемы можно произвести отбор элементов из справочника "Клиенты" по группам клиентов, региону, дополнительным свойствам и т.д, т.е. по всем имеющимся реквизитам у клиента. 
  • По взаимодействиям При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанными с взаимодействиями с клиентом. Используя эти показатели можно например, сформировать сегмент по тем клиентам, с которыми давно не было взаимодействий для возобновления с ними отношений.
    Важно! При использовании данной схемы можно выбрать - по каким документам взаимодействий проводить анализ. Для этого используется отбор по полю "Группа универсального журнала". Для отбора взаимодействий можно использовать преднастроенную группу "Взаимодействия", но так же можно создать свои группы с нужным набором видов документов в нем. Справочник "Группы универсального журнала" доступен в разделе "Настройки" в группе "НСИ".
  • По продажам При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанным с продажей товаров/услуг клиентам: выручка, количество проданной номенклатуры, вес позиции (номенклатуры), общий вес клиента. Используя эти показатели можно отслеживать выполнение условий продаж партнерами (например, сумма выручка за квартал больше 10000 рублей.
  • По расчетам При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанным с взаиморасчетами с клиентами: долг клиента и наш долг, срок задолженности клиента. Используя эти показатели можно отслеживать список тех клиентов, которые должны нам большие суммы и тех поставщиков, по которым есть задолженность. Тех клиентов, которые превысили установленный на предприятии максимальный кредит задолженности можно отнести к сегменту "Продажи запрещены".

Важно! При формировании сегментов клиентов можно также использовать любые произвольные запросы. Для формирования запросов используется схема компоновки данных.
